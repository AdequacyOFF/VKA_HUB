# VKAHUB Backend

Backend на FastAPI с Чистой Архитектурой для реализации платформы управления соревнованиями.

## Архитектура

Проект следует принципам Чистой Архитектуры (Clean Architecture):

- **Domain Layer (Слой домена)**: Бизнес-сущности, модели и интерфейсы репозиториев
- **Use Cases (Сценарии использования)**: Бизнес-логика и правила, специфичные для приложения
- **Infrastructure (Инфраструктура)**: Внешние зависимости (база данных, хранилище файлов, безопасность)
- **Presentation (Представление)**: API конечные точки, DTOs и обработка запросов/ответов

## Структура

```
app/
├── config/              # Конфигурация приложения
├── domain/              # Основная бизнес-логика
│   ├── entities/        # Бизнес-сущности
│   ├── models/          # ORM модели SQLAlchemy
│   └── repositories/    # Интерфейсы репозиториев
├── use_cases/           # Правила бизнес-логики приложения
│   ├── auth/            # Сценарии аутентификации
│   ├── user/            # Управление пользователями
│   ├── teams/           # Управление командами
│   ├── competitions/    # Управление соревнованиями
│   ├── certificates/    # Управление сертификатами
│   ├── reports/         # Генерация отчётов
│   └── moderators/      # Управление модераторами
├── infrastructure/      # Внешние интеграции
│   ├── db/              # Подключение к базе данных
│   ├── repositories/    # Реализации репозиториев
│   ├── security/        # Аутентификация и безопасность
│   └── storage/         # Обработка файлов
└── presentation/        # Слой API
    ├── api/             # API маршруты и DTOs
    └── middlewares/     # Middleware для запросов/ответов
```

## Установка

### Локальная разработка

1. Создать виртуальное окружение:
```bash
python -m venv venv
source venv/bin/activate  # В Windows: venv\Scripts\activate
```

2. Установить зависимости:
```bash
pip install -r requirements.txt
```

3. Настроить переменные окружения:
```bash
cp .env.template .env
# Отредактируйте .env с вашей конфигурацией
```

4. Выполнить миграции:
```bash
alembic upgrade head
```

5. Запустить сервер:
```bash
uvicorn app.main:app --reload
```

API будет доступен по адресу http://localhost:8000

### Документация API

- Swagger UI: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

## Миграции базы данных

Создать новую миграцию:
```bash
alembic revision --autogenerate -m "описание"
```

Применить миграции:
```bash
alembic upgrade head
```

Откатить:
```bash
alembic downgrade -1
```

## Тестирование

Запустить все тесты:
```bash
pytest
```

Запустить с покрытием кода:
```bash
pytest --cov=app --cov-report=html
```

## Возможности

- Аутентификация на основе JWT с refresh токенами
- Контроль доступа на основе ролей (Пользователь, Капитан, Модератор)
- Обработка загрузки файлов с валидацией
- Генерация отчётов .docx
- Журналирование активности
- Пагинация и фильтрация
- Асинхронные операции с базой данных

## API Конечные точки

Полную справку по API см. в документации Swagger по адресу `/docs`.

### Основные конечные точки

- **Аутентификация**: `/api/auth/*`
- **Пользователи**: `/api/users/*`
- **Команды**: `/api/teams/*`
- **Соревнования**: `/api/competitions/*`
- **Сертификаты**: `/api/certificates/*`
- **Отчёты**: `/api/reports/*`
- **Модератор**: `/api/moderator/*`
